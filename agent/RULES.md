# /agent/RULES.md

## Agent Rules for BloomSpace

You are helping build BloomSpace v1.

## Main Rule

Do not overbuild.

Stay focused on the MVP.

## Before Every Change

Read:

* /agent/GOAL.md
* /agent/STATUS.md
* /agent/NEXT.md
* /agent/RULES.md

Then say:

1. Current goal
2. Current phase
3. Next task
4. Files you plan to change

## During Coding

* Make small changes
* Keep code simple
* Prefer working app over perfect architecture
* Do not add features outside MVP
* Do not install packages unless needed
* Do not refactor unrelated files
* Do not create complex abstractions too early

## After Every Change

Update:

* /agent/STATUS.md
* /agent/NEXT.md

Then report:

1. What changed
2. What works now
3. What still needs work
4. What command the user should run next

## Product Principles

BloomSpace should feel:

* Calm
* Simple
* Encouraging
* Non-judgmental
* ADHD-friendly

Avoid language that feels harsh, stressful, or productivity-shaming.
